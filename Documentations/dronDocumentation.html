Project Introduction

This document details the design, assembly, and programming of a quadcopter Unmanned Aerial Vehicle (UAV), developed using the ArduPilot open-source platform.

The primary objective of this project is to create a stable platform capable of autonomous GPS navigation and real-time telemetry. The system is built around a Pixhawk flight controller, integrating inertial sensors and global positioning systems.

Basic initialization command in the MAVLink console:

MAV> arm throttle
MAV> takeoff 10

The project demonstrates competencies in power electronics, control systems, and embedded firmware configuration.

Core Hardware

The core of the system is the Flight Controller (FC), which manages stabilization and navigation. Modular hardware was selected to allow for future expansions.

Flight Controller: Pixhawk 2.4.8 (32-bit ARM Cortex M4)
GPS/Compass: High-precision Ublox M8N Module
Radio Control: FlySky FS-i6X with FS-iA6B receiver (iBUS Protocol)
Propulsion System

Connection Architecture

The electronic architecture follows a centralized power distribution scheme using a PDB (Power Distribution Board).

The main data flow occurs between the RC receiver and the Pixhawk via the RCIN port, while telemetry is managed through UART serial ports.

VCC = 11.1V (3S LiPo)
Signal = PWM / PPM / iBUS
V_Sensor = Active voltage and current monitor

Critical sensors (Accelerometer, Gyroscope, Barometer) interact directly with the firmware's EKF (Extended Kalman Filter) to estimate the vehicle's attitude.

GPS: M8N Module (Serial + I2C)
MAIN OUT 1-4: Signal to ESCs

Firmware Configuration

The system runs on ArduCopter 4.x. Initial configuration requires firmware flashing and setting mandatory parameters via Mission Planner.

Critical Parameters

To ensure safe takeoff, variables in the full parameter tree have been modified. It is vital to configure the battery and radio "Failsafe".

ARMING_CHECK = 1 (Enables all safety checks)
FLTMODE1 = Stabilize
BATT_MONITOR = 4

Accelerometer calibration requires placing the drone in 6 different positions to register MEMS sensor offsets.

Development Challenges

During the integration phase, the biggest challenge has been calibrating the internal and external compass due to electromagnetic interference (EMI) generated by high-current ESC cables.

Work is currently underway on tuning the PID control loops to improve stability:

P (Proportional): Immediate reaction to error.
I (Integral): Correction of accumulated long-term error.
D (Derivative): Dampening of future oscillations.

The drone exhibits minor oscillations in the "Roll" axis during aggressive maneuvers, indicating a 'P' value that is too high or insufficient 'D'.

Current Status & Results

The drone has performed successful takeoffs in "Stabilize" mode. The GPS system achieves a "3D Fix" with over 12 satellites, enabling assisted modes.

List of pending tasks for optimization:

  • PID Gain fine-tuning (AutoTune).
  • Dynamic propeller balancing to reduce vibrations.
  • Geo-Fence configuration.
  • MavLink telemetry implementation.
  • "Return to Launch" (RTL) testing.
  • Integration of a camera with a 2-axis Gimbal.
