Introduction & Achievements

Flappy Iron Man is an advanced reinterpretation of the classic arcade genre, developed in Python utilizing the Pygame engine. This project extends beyond basic physics and collision mechanics by introducing an offensive combat system and dynamic environment destruction.

The project was collaboratively developed by a team consisting of "aserrajjj", "susda", and myself. Following a technical evaluation and gameplay showcase, this software was awarded:

The game features a stable 60 FPS runtime, integrated video playback, and a progressive difficulty algorithm.

Technology Stack

The graphics engine core is built on Pygame, implementing an optimized Game Loop to handle event listeners, state updates, and real-time rendering.

Key libraries imported in main.py:

import pygame # Graphics and Audio Engine
import os # Asset Path Management
import random # Procedural Level Generation
from files.pyvidplayer import Video # Custom Video Codec
Programming Paradigm

A strict Object-Oriented Programming (OOP) approach was utilized. Every game entity is an independent class inheriting from pygame.sprite.Sprite, ensuring code scalability and maintainability.

Project Architecture

The file structure is modularized to separate game logic, assets, and class definitions. The entry point is main.py, which orchestrates the following modules:

/files/character_class.py: Player Physics (Iron Man).
/files/wand_class.py: Obstacle Generation (Walls).
/files/buttons_class.py: User Interface (UI).
/files/score.py: Score Management & Persistence.

The system uses "Sprite Groups" for efficient collision handling and batch rendering within the draw_window() function.

Mechanics & Design

Unlike the original game, Flappy Iron Man introduces strategic offensive mechanics. The player must balance evasion with environment interaction.

Character: Iron Man (Simulated Gravity Flight)
Resource (Coins): Converted into Ammunition upon collection.
Combat System: Repulsor shots to destroy obstacles.
Durability: Walls require 2 hits to break.
Resolution: 1200 x 600 px

This duality (Dodge vs. Destroy) adds a layer of strategic depth that was decisive in securing the competition victory.

Key Algorithms

A major challenge was the Progressive Difficulty Algorithm. The spawn frequency of walls increases mathematically based on the player's current score.

# Obstacle Generation Formula
if time() - Begin_time > ((1.2) - (0.025 * variables.score)):
add_walls()
# Higher score = Lower time between walls

Pixel-perfect (or AABB) collision detection was implemented using the spritecollide method. This manages both defeat conditions (crashing into a wall) and resource gathering (collecting coins).

Project Results

The final software is robust and free of memory leaks thanks to correct sprite disposal (.kill() method) when objects leave the screen. The integration of an intro video via pyvidplayer adds high production value.
