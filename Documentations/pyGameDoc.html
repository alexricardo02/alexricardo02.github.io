1. Introduction

Ironfly is a game developed with Pygame, inspired by Flappy Bird, but with an additional mechanic: the player can shoot after collecting coins. It is a single-player game where the goal is to score as many points as possible by avoiding obstacles and destroying walls.

This project was developed as part of a university computer science practice in September 2022. The team consisted of three members: David, Adam, and myself.

2. Main Features

  • Genre: Arcade
  • Game Mode: Single player
  • Objective: Earn the highest score by dodging obstacles and shooting when possible.
  • Core Mechanics:
    • The player jumps by clicking or tapping the screen.
    • Coins can be collected to enable shooting and destroying walls.
  • Visual Style: Arcade, futuristic design.
  • Sound:
    • Arcade-style background music.
    • Sound effects for jumping and wall destruction.

3. Installation and Execution

Prerequisites

Make sure you have Python installed (recommended version: 3.x) and install the Pygame library:

pip install pygame

Running the Game

Clone the repository and run the game:

git clone https://github.com/alexelricardo02/FlappyBird_EIP_Praktikum.git
cd FlappyBird_EIP_Praktikum
python main.py

4. Code Architecture

The project is structured in a modular fashion to enhance readability and maintainability. The codebase is organized as follows:

  • Main Module: Contains the game loop and overall game state management.
  • Game Entities: Separate classes and modules for the player (the bird), obstacles, coins, and shooting mechanics.
  • Event Handling: Dedicated functions for processing user input (mouse clicks or screen taps) and managing game events.
  • Asset Management: A directory structure to organize graphics, sounds, and other media resources.

This separation of concerns allows for easier maintenance and potential expansion of game features.

5. Development and Technical Challenges

During the development of Ironfly, several technical challenges were addressed:

  • Timing and Frame Rate: Optimizing the game loop to ensure smooth gameplay and synchronization of frame rates.
  • Collision Detection: Implementing accurate collision detection between the bird, obstacles, and coins to maintain a responsive gaming experience.
  • Shooting Mechanic Integration: Balancing the shooting functionality (enabled after coin collection) with the core gameplay without disrupting the game flow.
  • Asset Integration: Coordinating the loading and management of visual and audio assets to complement the arcade aesthetic.
  • Time Constraints: Completing the project within a one-month period required efficient planning, collaboration, and rapid iteration.
